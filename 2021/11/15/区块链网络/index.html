区块链网络

发表于 2021-11-15

接下来会在概念层面上描述 Hyperledger Fabric 是如何让组织间以区块链网络的形式进行合作的。

什么是区块链网络

区块链网络是一个为应用程序提供账本和智能合约服务的技术基础设施。
首先，智能合约被用来生成交易，接下来这些交易会被分发给网络中的每个节点，这些交易会被记录在他们的账本副本上且不可篡改。
一般多个组织聚集在一起作为一个联盟来形成一个网络。

创建网络

让我们从头开始创建网络的基础吧！

图中有四个元素：

  • 排序服务O4
  • 网络配置NC4
  • 组织R4
  • 证书颁发机构CA4

当一个排序服务启动后就形成了如图的网络N。其中排序服务o4由一个单独的节点组成，是根据一个网络配置NC4来进行配置的。证书颁发机构CA4被用来向管理员和组织R4的网络节点分配身份信息（具体后面会讲到）

可以看到定义网络N的时候，第一件事就是定义一个排序服务。O4 最初被配置并且由组织 R4 的一个管理员来启动，并且由 R4 管理，配置 NC4 包含了描述网络管理能力初始集合的规则。最初在网络中集合仅赋予了 R4 这个权利。
目前 R4 是这个网络中唯一的一个成员

添加网络管理员

目前 NC4 最初被配置为仅仅允许 R4 用户在网络中具有管理的权限,接下来，允许R1用户也具有管理的权限。

新增元素： 组织R1即它的证书颁发机构CA1

组织 R4 更新了网络配置来使组织 R1 也成为了管理员。现在 R1 和 R4 在网络配置中便具有了相同的权限。
证书颁发机构 CA1 也被添加进来了，他用来标识 R1 组织的用户。现在从 R1 和 R4 来的用户就已经是网络的管理员了。

定义联盟

新增元素：联盟X1即其中的联盟成员——组织R1,R2、对应CA

  • 网络管理员定义了一个包含两个成员的联盟 X1，包含组织R1 和 R2 ,CA1 和 CA2 是这两个组织对应的证书颁发机构。
  • 这个联盟的定义被存储在了网络配置 NC4 中，会在接下来的网络开发中被使用。
    为什么联盟这么重要？我们能够看到联盟定义了网络中的一部分组织，他们共享了彼此能够交易的需求，在这个示例中就是 R1 和 R2 能够进行交易。

为联盟创建通道

通道——联盟中的成员彼此进行通信的主要机制。在一个网络中可能会有多个通道。

新增元素：通道C1，通道配置CC1

第一个通道是如何被添加到这个网络中的呢？

  • 通道C1——联盟X1为R1和R2创建
  • 通道配置CC1——控制管理通道C1，且完全独立于网络配置
  • CC1 是由 R1 和 R2 管理的，他们在 C1 上具有同等的权利。R4 在 CC1 中是没有任何权利的。因为他不是联盟成员
  • 该通道仅能处理在 R1 和 R2 之间进行的交易
  • 一个通道可以包含任意数量的组织，目前只包含了两个组织R1,R2

通道的重要性

  • 提供了一个联盟成员之间进行私有通信和私有数据的机制。实现了Fabric允许组织间共享基础设施的同时又保持了私有性。好比QQ应用（区块链网络）中的一个群聊（联盟），不同群聊间信息隐蔽不可见。
  • 通道实现了“从网络中解放出来”——网络配置 NC4 的任何改动都不会对通道配置 CC1 造成任何直接的影响，只有通道配置中指定的组织(QQ群聊认定的管理员)才能控制它

节点和账本

接下来使用通道将区块链网络以及组织的组件关联到一起

新增元素：节点P1，账本L1

  • Peer节点P1加入了通道C1。
  • P1存储账本L1的副本
  • P1和O4使用通道C1进行通信

可以发现P1与CA1同为紫色，表明是CA1为P1颁发的身份信息，他将P1和组织R1关联起来。P1启动后，它就向排序服务O4发出请求加入通道C1。O4收到加入请求，就会使用通道配置CC1来决定P1在这个通道的权限，比如：CC1 决定 P1 是否能够向账本 L1 中读取或写入信息。

应用程序和智能合约链码

现在通道C1拥有了一个账本，我们可以连接客户端应用来使用由 Peer 节点提供的服务了。

新增元素：智能合约S5，客户端应用A1

  • 智能合约S5被安装在P1
  • 在组织R1中的客户端应用A1可以通过P1使用S5来访问账本
  • A1、P1、O4都加入了通道C1，他们都可以使用这个通道提供的通信设备
  • 尽管客户端应用处在Fabric区块链网络的外边，但它是可以通过通道C1跟网络相连的

就像Peer节点和排序节点一样，客户端应用也会有一个是它和组织相关联的身份信息

安装链码包

智能合约会被打包进一个链码中，这个链码会被部署到一个区块链网络中。可以把智能合约想象为管理交易，链码则管理着智能合约应该如何被打包部署。

在智能合约S5被开发完后，组织 R1 中的管理员必须要把它安装到节点 P1 上，当完成之后，P1 就完全了解了 S5。
当一个组织在一个通道中有多个 Peer 节点时，可以选择在哪个节点安装智能合约，而不需要每个 Peer 节点上都安装。
