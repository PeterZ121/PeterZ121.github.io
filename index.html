Fabric区块链--成员服务提供者(MSP)

发表于 2021-11-15

为什么需要MSP?

  • Fabric是一个认证性的网络, 所以区块链参与者需要一种向网络中的其他参与者证实自己身份的机制从而在网络中进行交易。
  • 证书颁发机构就像一个信用卡提供商(CA)，它分发许多不同类型的可验证的身份(数字证书)。另一方面，MSP确定商店接受哪些信用卡提供商(哪些CA)。通过这种方式，MSP将一个身份(信用卡)转换为一个角色(在商店购买东西的能力)。


Identities类似于你的信用卡，用来证明你可以支付。MSP类似于被商店接受的信用卡清单。

  • 此时你想要加入一个现有的网络，你需要一种方法——MSP来将你的身份转换为可以被网络识别的东西
  • 要想加入区块链网络，在Fabric网络上交易，需要：
    1. 拥有一个由网络信任的CA颁发的身份
    2. 成为一个被认可的成员（成员资格是通过将该成员的公钥添加到组织的MSP来实现的）
    3. 将MSP添加到网络上的一个联盟或者通道
    4. 确保MSP包括在网络中的策略定义

MSP的功能

  • MSP通过列出其成员的身份，或通过确定哪些是为其成员授权颁发有效身份的ca，来识别和确定接受来自这些根ca和中间ca所定义的信任域的成员。
  • MSP通过标识参与者在节点或通道上拥有的特定特权，将身份转换为角色
  • MSP可以识别被撤销的身份列表。

MSP域

分为

  • 本地MSP：在参与者节点本地
  • 通道MSP：在通道配置中

共同点：都是将身份转化为角色
区别：它们的范围

组织与MSP

  • 是一个逻辑上成员们的管理组
  • 在单个MSP下管理其成员
  • 一个组织可以被划分为多个**组织单元(ou)**。如果不使用偶，MSP中的所有身份（由根CA和中间CA文件夹认定的)都将被视为组织的成员。

重点掌握什么是PKI

什么是身份？

  • 区块链网络中的不同参与者都具有封装在 X.509数字证书 中的 数字身份 。它们确定了 对资源的确切权限 以及对参与者在区块链网络中拥有的信息的访问权限
  • 主体——决定身份权限的属性 包含了参与者的身份的各种属性，如参与者的组织、组织单位，角色甚至是参与者的特定身份。
  • 身份验证

什么是PKI？

  • 公钥基础结构（PKI）是一组互联网技术，可在网络中提供安全通信。如：PKI使HTTP变成HTTPS.
  • 区块链网络依赖于PKI标准来确保各个网路参与者之间的安全通信，并确保在区块链上发布的信息得到适当的认证
  • 四个关键要素
    1. 数字证书
    2. 公钥和私钥
    3. 证书授权中心
    4. 证书撤销列表

数字证书

  • 如一个人的身份证（提供了本人信息），其公钥是在证书中分发的，签名密钥则保密。
  • 若被篡改则会使证书无效

授权，公钥和私钥

  • 密钥之间唯一的数学关系使得私钥在消息上的签名，只有对应公钥在相同的消息上才可以与之匹配。
  • 签名及验证流程如下


证书授权中心(CA)

  • 人员或节点能够通过由系统信任的机构为其发布的数字身份参与区块链网络。
  • 最常见的身份形式：符合X.509标准 并 由CA颁发的经加密验证的数字证书

    根CA,中间CA和信任链

    Fabric CA

  • Fabric提供了一个内置的CA组件——Fabric CA，允许在自己的区块链网络中创建CA

证书撤销列表（CRL）

  • 是 CA 知道由于某些原因而被撤销的证书的引用列表。抽象为：商店场景中的被盗信用卡列表
  • 用CPL检查证书是否有效：如果模仿者试图将无效的数字证书传递给验证者，则可以首先检查颁发证书的 CA 的 CRL，以确保其未被列为无效。

区块链网络

发表于 2021-11-15

接下来会在概念层面上描述 Hyperledger Fabric 是如何让组织间以区块链网络的形式进行合作的。

什么是区块链网络

区块链网络是一个为应用程序提供账本和智能合约服务的技术基础设施。
首先，智能合约被用来生成交易，接下来这些交易会被分发给网络中的每个节点，这些交易会被记录在他们的账本副本上且不可篡改。
一般多个组织聚集在一起作为一个联盟来形成一个网络。

创建网络

让我们从头开始创建网络的基础吧！

图中有四个元素：

  • 排序服务O4
  • 网络配置NC4
  • 组织R4
  • 证书颁发机构CA4

当一个排序服务启动后就形成了如图的网络N。其中排序服务o4由一个单独的节点组成，是根据一个网络配置NC4来进行配置的。证书颁发机构CA4被用来向管理员和组织R4的网络节点分配身份信息（具体后面会讲到）

可以看到定义网络N的时候，第一件事就是定义一个排序服务。O4 最初被配置并且由组织 R4 的一个管理员来启动，并且由 R4 管理，配置 NC4 包含了描述网络管理能力初始集合的规则。最初在网络中集合仅赋予了 R4 这个权利。
目前 R4 是这个网络中唯一的一个成员

添加网络管理员

目前 NC4 最初被配置为仅仅允许 R4 用户在网络中具有管理的权限,接下来，允许R1用户也具有管理的权限。

新增元素： 组织R1即它的证书颁发机构CA1

组织 R4 更新了网络配置来使组织 R1 也成为了管理员。现在 R1 和 R4 在网络配置中便具有了相同的权限。
证书颁发机构 CA1 也被添加进来了，他用来标识 R1 组织的用户。现在从 R1 和 R4 来的用户就已经是网络的管理员了。

定义联盟

新增元素：联盟X1即其中的联盟成员——组织R1,R2、对应CA

  • 网络管理员定义了一个包含两个成员的联盟 X1，包含组织R1 和 R2 ,CA1 和 CA2 是这两个组织对应的证书颁发机构。
  • 这个联盟的定义被存储在了网络配置 NC4 中，会在接下来的网络开发中被使用。
    为什么联盟这么重要？我们能够看到联盟定义了网络中的一部分组织，他们共享了彼此能够交易的需求，在这个示例中就是 R1 和 R2 能够进行交易。

为联盟创建通道

通道——联盟中的成员彼此进行通信的主要机制。在一个网络中可能会有多个通道。

新增元素：通道C1，通道配置CC1

第一个通道是如何被添加到这个网络中的呢？

  • 通道C1——联盟X1为R1和R2创建
  • 通道配置CC1——控制管理通道C1，且完全独立于网络配置
  • CC1 是由 R1 和 R2 管理的，他们在 C1 上具有同等的权利。R4 在 CC1 中是没有任何权利的。因为他不是联盟成员
  • 该通道仅能处理在 R1 和 R2 之间进行的交易
  • 一个通道可以包含任意数量的组织，目前只包含了两个组织R1,R2

通道的重要性

  • 提供了一个联盟成员之间进行私有通信和私有数据的机制。实现了Fabric允许组织间共享基础设施的同时又保持了私有性。好比QQ应用（区块链网络）中的一个群聊（联盟），不同群聊间信息隐蔽不可见。
  • 通道实现了“从网络中解放出来”——网络配置 NC4 的任何改动都不会对通道配置 CC1 造成任何直接的影响，只有通道配置中指定的组织(QQ群聊认定的管理员)才能控制它

节点和账本

接下来使用通道将区块链网络以及组织的组件关联到一起

新增元素：节点P1，账本L1

  • Peer节点P1加入了通道C1。
  • P1存储账本L1的副本
  • P1和O4使用通道C1进行通信

可以发现P1与CA1同为紫色，表明是CA1为P1颁发的身份信息，他将P1和组织R1关联起来。P1启动后，它就向排序服务O4发出请求加入通道C1。O4收到加入请求，就会使用通道配置CC1来决定P1在这个通道的权限，比如：CC1 决定 P1 是否能够向账本 L1 中读取或写入信息。

应用程序和智能合约链码

现在通道C1拥有了一个账本，我们可以连接客户端应用来使用由 Peer 节点提供的服务了。

新增元素：智能合约S5，客户端应用A1

  • 智能合约S5被安装在P1
  • 在组织R1中的客户端应用A1可以通过P1使用S5来访问账本
  • A1、P1、O4都加入了通道C1，他们都可以使用这个通道提供的通信设备
  • 尽管客户端应用处在Fabric区块链网络的外边，但它是可以通过通道C1跟网络相连的

就像Peer节点和排序节点一样，客户端应用也会有一个是它和组织相关联的身份信息

安装链码包

智能合约会被打包进一个链码中，这个链码会被部署到一个区块链网络中。可以把智能合约想象为管理交易，链码则管理着智能合约应该如何被打包部署。

在智能合约S5被开发完后，组织 R1 中的管理员必须要把它安装到节点 P1 上，当完成之后，P1 就完全了解了 S5。
当一个组织在一个通道中有多个 Peer 节点时，可以选择在哪个节点安装智能合约，而不需要每个 Peer 节点上都安装。

Fabric区块链关键概念总结

发表于 2021-11-15

1. 简记

第一个广为人知的区块链应用是比特币，以太坊是另一种加密货币，采用不同的办法，新增了智能合约为分布式应用创建了一个平台。比特不和以太坊属于同一类区块链：公共非许可区块链技术，允许任何人在上面匿名互动


2. Hyperledger Fabric

Hyperledger(超级账本) Fabric是开源的企业级许可分布式账本技术平台

  1. 具有模块化和可配置的架构
  2. 第一个支持通用编程语言编写智能合约（如Java Go Node.js)
  3. Fabric平台参与者彼此了解而非匿名（与公共非许可网络的不同）
  4. 支持可插拔的共识协议，使平台能够更有效地进行定制

3. 为什么说Fabric是模块化架构

Fabric由以下模块化的组件组成：

  • 可插拔的排序服务——对交易顺序建立共识，然后向节点广播区块；
  • 可插拔的**成员服务提供者(MSP)**——将网络中的实体与加密身份相关联
  • 可选的P2P gossip 服务——将区块发送到其他结点（通过排序服务实现）
  • 智能合约（链码）——隔离运行在容器环境中（例如在Docker中）
  • 账本——可以通过配置支持多种DBMS
  • 可插拔的背书和验证策略——每个应用程序可以独立配置。

没有“可以一统天下的链（one blockchain to rule them all）”。Hyperledger Fabric 可以通过多种方式进行配置，以满足不同行业应用的需求。

4. 学习Fabric前需要知道的概念

  1. 共享账本

    Hyperledger Fabric 有一个账本子系统，包括两个组件： 世界状态和交易日志。
    世界状态组件是账本的数据库，交易日志组件记录产生世界状态中当前值的所有交易。账本包括世界状态数据库和交易日志历史记录。

  2. 智能合约

    Hyperledger Fabric 智能合约用于实现一整套账本功能（交易、查询等），例如，可以编写智能合约以规定运输物品的成本，其中运费根据物品到达的速度而变化。根据双方同意并写入账本的条款，当收到物品时，相应的资金会自动转手。
    智能合约用链码编写，链码支持Go、Nose.js、Java编写

  3. 共识

    保持账本在整个网络中同步的过程称为 共识 。该过程确保账本仅在交易被相应参与者批准时更新，并且当账本更新时，它们以相同的顺序更新相同的交易。
