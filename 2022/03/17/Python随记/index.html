获取命令行参数

1. getopt模块

  • 解析-p/-a等参数

  • args: 要解析的命令行参数列表。

  • options : 以字符串的格式定义，存放短参数，options 后的冒号 : 表示如果设置该选项，必须有附加的参数，否则就不附加参数。如：”ab:c:d”，表示参数-a，-d不附加参数，-b、-c附加参数

  • long_options : 以列表的格式定义，存放长参数，long_options 后的等号 = 表示该选项必须有附加的参数，不带等号表示该选项不附加参数。如：[“ifile=”,”ofile”]，表示—ifile附加参数，—ofile不带参数

  • 返回值由两个元素组成: 第一个是 (option, value) 元组的列表：[(option, value),(option,value),(参数,附加参数)]。 第二个是参数列表，包含那些没有 - 或 — 的参数。

字符串、数字以及进制转换

字符串操作

大小写相关：

  • capitalize() 返回首个字符大写，其余为小写的字符串副本。
  • casefold() 返回原字符串消除大小写的副本
  • lower() 返回原字符串的副本，其所有区分大小写的字符均转换为小写。
  • upper() 返回原字符串的副本，其中所有区分大小写的字符均转换为大写。
  • title() 返回原字符串的标题版本，其中每个单词第一个字母为大写，其余字母为小写。

字符串裁剪

  • strip([chars]) 返回原字符串的副本，移除其中的前后字符。 chars 参数为指定要移除字符的字符串。 如果省略或为 None，则 chars 参数默认移除空白符。另有 lstrip() 方法移除其前导字符，rstrip() 方法移除其结尾字符。
  • split(sep=None) 返回一个由字符串内单词组成的列表，使用 sep 作为分隔字符串。

字符串检查

  • startswith(prefix) 如果字符串以指定的 prefix 开始则返回 True，否则返回 False。
  • endswith(suffix) 如果字符串以指定的 suffix 结束返回 True，否则返回 False。
  • isalnum() 如果字符串中的所有字符都是字母或数字且至少有一个字符，则返回 True ， 否则返回 False 。
  • isalpha() 如果字符串中的所有字符都是字母，并且至少有一个字符，返回 True ，否则返回 False 。
  • islower() 如果字符串中至少有一个区分大小写的字符 4 且此类字符均为小写则返回 True ，否则返回 False 。
  • isupper() 如果字符串中至少有一个区分大小写的字符 4 且此类字符均为大写则返回 True ，否则返回 False 。

JSON

  • 解码load: json字符串转对象

  • 编码dump: 对象转json字符串

方法 功能
json.dump(obj,fp) 将python 数据类型转换为json格式的字符串并保存到json格式的fp文件中
json.dumps(obj) 将Python数据类型转换为json格式的字符串
json.load(fp) 从json格式的文件中读取json格式的字符串并转换为Python对应类型
json.loads(str) 将json格式的字符串转换为python对应类型

1. dump的使用

  • 常用参数：

    • indent: 设置缩进格式，默认值为None,选择的是最紧凑的表示。如果indent是非负整数或字符串，那么JSON数组元素和对象成员将使用该缩进级别进行输入；indent为0、负数或“ ”仅插入换行符；indent使用正整数缩进多个空格；如果indent是一个字符串（例如“\t”），则该字符串用于缩进每个级别。

  • separators: 去除分隔符后面的空格，如果指定，则分隔符应为（item_separator，key_separator）元组。如果缩进为None，则默认为（’，’，’：’）;要获得最紧凑的JSON表示，可以指定（’，’，’:’）以消除空格。

  • sort_keys: 如果sort_keys为True，则字典的输出将按键值排序。

  • skipkeys: 如果skipkeys=True, 则将跳过不是基本类型（str，int，float，bool，None）的dict键，不会引发TypeError。

  • ensure_ascii: 能将所有传入的非ASCII字符转义输出。如果ensure_ascii为False，则这些字符将按原样输出。

  • check_circular: 如果check_circular为False，则将跳过对容器类型的循环引用检查，循环引用将导致OverflowError。

  • allow_nan: 如果allow_nan为False，则严格遵守JSON规范,序列化超出范围的浮点值（nan，inf，-inf）会引发ValueError。 如果allow_nan为True,则将使用它们的JavaScript等效项（NaN，Infinity，-Infinity）。

  • default: 如果指定，则default应该是为无法以其他方式序列化的对象调用的函数。它应返回对象的JSON可编码版本或引发TypeError。如果未指定，则引发TypeError。

CSV文件读写

  • CSV 格式是电子表格和数据库中最常见的输入、输出文件格式

1. csv.reader(csvfile, dialect=’excel’, **fmtparams)

  • 返回值：一个 reader （阅读器）对象，是一个List，该对象将逐行遍历 csvfile。
  • 参数：
    • csvfile，必须是支持迭代(Iterator)的对象，可以是文件(file)对象或者列表(list)对象
    • dialect，编码风格，默认为excel的风格，也就是用逗号（,）分隔，dialect 方式也支持自定义，通过调用register_dialect方法来注册
    • fmtparam，格式化参数，用来覆盖之前dialect对象指定的编码风格。

2. csv.writer(csvfile, dialect=’excel’, **fmtparams)

  • 首先open()函数打开当前路径下的名字为’t.csv’的文件，如果不存在这个文件，则创建它，返回myFile文件对象。

  • csv.writer(myFile)返回writer对象myWriter。

  • writerow()方法是一行一行写入，writerows方法是一次写入多行。
  • 注意：如果文件’t.csv’事先存在，调用writer函数会先清空原文件中的文本，再执行writerow/writerows方法。

3. register_dialect(name, [dialect, ]**fmtparams)

  • 自定义dialect,将 dialect 与 name 关联起来。 name 必须是字符串
  • name,你所自定义的dialect的名字，比如默认的是’excel’，你可以定义成’mydialect’
