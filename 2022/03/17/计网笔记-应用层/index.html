计网笔记-应用层

发表于 2022-03-17 更新于 2022-03-21

一、应用层概述

1. 两种应用程序体系结构

  1. client-service(C/S,B/S)

    • 服务器具有固定、周知的IP地址（应该说是域名固定，IP会变化以保证安全）

  2. peer-to-peer(P2P)

    • 任意终端之间可直接通信
    • 对等节点从其他对等节点请求服务，同时可以为其他对等节点提供服务
    • 自我可扩展性

2. 进程寻址

​ 进程有一个标识符，包含主机IP地址和端口号。

​ 创建一个新的网络应用程序的进程时，必须分配一个新的端口号。不与公开的端口号（0-1023）或本机已使用的端口号重复。

3. 应用程序使用的传输服务有

  • 数据完整性

  • 吞吐量保证

  • 时间保证

  • 安全保证

二、HTTP

1. 整个Web访问过程

(1)浏览器发送一个请求到服务端，去请求所需要的相关资源;
(2)资源分为动态资源和静态资源,动态资源可以是使用Java代码按照Servlet和JSP的规范编写的内容，或javascript
(3)在Java代码可以进行业务处理也可以从数据库中读取数据;
(4)拿到数据后，把数据交给HTML页面进行展示,再结合CSS和JavaScript使展示效果更好;
(5)服务端将静态资源响应给浏览器;
(6)浏览器将这些资源进行解析;
(7)解析后将效果展示在浏览器，用户就可以看到最终的结果。

2. HTTP特点

  • 基于TCP协议：面向连接，安全

    1. 三次握手，四次挥手

    2. TCP提供但UDP不提供的服务：

      • 可靠传输：无差错的交付数据（若数据损失会重传数据，因此不能保证时延可控）
      • 流量控制：点对点的通信量的控制，即接收端控制发送端——抑制发送端发送数据的速率，以便接收端来得及接收。
      • 拥塞控制：抑制发送端发送数据的速率，以减轻所在网络负荷，全局性。
      • 时延可控
      • 尽力分发：不保证发送的数据能否被收到

    3. SSL (Secure Sockets Layer) ：安全套接字

  • 基于请求-响应模型：一次请求对应一次响应

  • 无状态：

    • 优点：速度快
    • 缺点：多次请求间不能共享数据——》使用Cookie、Session解决。

  • HTTP连接

    • 非持续HTTP连接

    • 持续HTTP连接（默认方式下使用持续连接）

    • RTT(Round-Trip Time) 往返时延

      表示从发送端发送数据开始，到发送端收到来自接收端的确认所经历时间

3. HTTPS、SSL、TLS

  • SSL (Secure Sockets Layer) ：安全套接字
  • TLS：TLS就是SSL的新版本3.1
  • HTTPS：HPPTS是HTTP+SSL/TCP的简称，并不是一个新的协议。易错

三、会话技术（Session和Cookie）

1. 什么是会话

1.1 会话：

​ 用户打开浏览器，访问web服务器的资源，会话建立，直到有一方断开连接，会话结束。在一次会话中可以包含多次请求和响应。

1.2 会话跟踪

​ 一种维护浏览器状态的方法，服务器需要识别多次请求是否来自于同一浏览器，以便在同一次会话的多次请求间共享数据。

  • 服务器会收到多个请求，这多个请求可能来自多个浏览器，如上图中的6个请求来自3个浏览器

  • 服务器需要用来识别请求是否来自同一个浏览器

  • 服务器用来识别浏览器的过程，这个过程就是会话跟踪

  • 服务器识别浏览器后就可以在同一个会话中多次请求之间来共享数据

2. 客户端会话跟踪技术：Cookie

客户端会话技术，将数据保存到客户端，以后每次请求都携带Cookie数据进行访问。

2.1 工作流程

  • 服务端提供了两个Servlet，分别是ServletA和ServletB

  • 浏览器发送HTTP请求1给服务端，服务端ServletA接收请求并进行业务处理

  • 服务端ServletA在处理的过程中可以创建一个Cookie对象并将name=zs的数据存入Cookie

  • 服务端ServletA在响应数据的时候，会把Cookie对象响应给浏览器

  • 浏览器接收到响应数据，会把Cookie对象中的数据存储在浏览器内存中，此时浏览器和服务端就建立了一次会话。在同一次会话中浏览器再次发送HTTP请求2给服务端ServletB，浏览器会携带Cookie对象中的所有数据。

  • ServletB接收到请求和数据后，就可以获取到存储在Cookie对象中的数据，这样同一个会话中的多次请求之间就实现了数据共享

3. 服务器会话跟踪技术：Session

3.1 概念

Session：服务端会话跟踪技术：将数据保存到服务端。

  • Session是存储在服务端而Cookie是存储在客户端
  • 存储在客户端的数据容易被窃取和截获，存在很多不安全的因素
  • 存储在服务端的数据相比于客户端来说就更安全

三、因特网中的电子邮件-SMTP

1. 三个组成部分

  • 用户代理 user agents：一个客户端程序
  • 邮件服务器 mail servers
  • 简单邮件传输协议 SMTP simple mail transfer protocol

2. 用户代理与邮件服务器

  • 当用户完成邮件撰写时，用户代理向其邮件服务器发送邮件，并存放在发送队列中。

  • 当用户想读取一条报文时，用户代理从其邮件服务器的邮箱中获取该报文。

3. SMTP

  • 使用TCP实现电子邮件消息从客户端到服务器的可靠传输, port 25,持续连接

  • 直接传输：发送服务器到接收服务器

  • 命令/响应交互(like HTTP, FTP)

    • ==commands==: ASCII 文本
    • ==response==: 状态码和短语
  • 消息都使用7位ASCII编码

  • 对于非ASCII字符文本，使用MIME协议

  • 使用==CRLF==（\r\n即回车并换行）作为消息的结束

4. 传输过程

5. SMTP与HTTP的对比

  • SMTP把所有报文的对象封装在一个响应报文中，而HTTP把每个对象各自封装一个响应报文

  • HTTP：拉模式。由客户端主动发起事务，向服务器提出申请，将所需资源从服务器“拉”回来

    SMTP：推模式。客户端告知服务器地址，服务器将信息推给客户端

6. POP3协议

  • 特点：简单，但功能有限
  • 工作步骤：
    • ==特许阶段==：用户代理发送用户名和口令获得下载邮件的特许。（身份认证）
    • ==事务处理阶段==：用户代理取回报文，可对邮件进行某些操作。如做删除标记、取消删除标记、获取统计信息等。
    • ==更新阶段==：邮件服务器删除带有删除标记的报文，结束POP会话。

四、DNS（Domain Name System）

1. 特点

  • 分布式数据库

  • 负载均衡

  • 主机别名（host aliasing)服务

2. 域名服务器层次

​ ==uestc.edu.cn==

2.1 根域名服务器

  • 维护顶级域名服务器的IP地址

  • 全世界就13个

  • 本地域名服务器如果无法解析，会向根域名服务器发起请求（极少发生）

2.2 顶级域名服务器

  • 维护顶级域名的IP地址
  • 如：com、org、net、edu、gov、jp、ca、cn

2.3 权威域名服务器

  • 由组织机构维护自己提供的服务器的域名的IP地址

2.4 本地域名服务器

  • 每个ISP都至少有一个本地域名服务器

  • 当主机发送DNS请求，这个请求会被发送给本地DNS服务器，本地DNS服务器会

    • 缓存最新收到的域名-地址的转换对(但是可能会过时!)

    • 作为代理，转发请求到层次化的DNS解析系统中

3. 域名解析方式

3.1 递归查询

不合理的查询，会导致根服务器流量过大

3.2 迭代查询

​ 根域名服务器收到本地域名服务器发出的迭代查询请求报文时，要么给出所要查询的IP地址，要么告诉本地域名服务器，“你下一步应当向哪一个顶级域名服务器进行查询”。然后让本地域名服务器进行后续的查询（而不是替本地域名服务器进行后续的查询）

4. DNS缓存

  • 在一个请求链中，当某DNS服务器接收一个DNS回答时，它能将映射缓存在本地存储器中。
    • 当TTL（Time To Live 生存时间）超时，缓存的映射条目会超时删除（通常为2天）
    • TLD服务器通常会被本地域名服务器缓存，以减少根域名服务器被访问次数
  • 如果主机的域名改变了IP地址，则缓存条目过时，当其所有缓存条目超时（TTL耗尽）时，因特网范围的所有域名服务器才算被更新
  • 更新/通知机制是在IETF标准中提出的——RFC 2136

5.DNS记录（RR）

DNS分布式数据库存储资源记录（resource record）

  • type=A

    • name=主机名，value=ip地址
    • 例子：(mjliu.ee.foo.com, 145.37.93.126, A)

  • type=NS

    • name=域名，value=ip地址

    • 例子：(foo.com, 1.2.3.4, NS)

    • 域名的范围要比主机名大。一个域名下可以有多个主机名，域名下还可以有子域名。例如，域名abc.com下，有主机server1和server2，其主机全名就是server1.abc.com和server2.abc.com。

  • type=CNAME

    • name=别名，value=规范主机名
    • 例子：(foo.com, mail.bar.foo.com, NS)

  • type=MX

    • name=别名，value=规范主机名

6. DNS报文

查询和应答报文，都是采用相同的报文格式
