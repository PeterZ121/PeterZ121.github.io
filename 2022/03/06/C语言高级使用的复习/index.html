一、命令行参数

  • argc：命令行参数的数量
  • argv：指向命令行参数的指针数组（数组元素为指针，指向一些字符串）。argv[0]指向程序名，argv[1~n-1]指向各参数。argv[argc]始终为空指针。最大的反对法

二、代码编译、程序组建

章节：15

  • 源文件：.c ；
  • 头文件：.h；存放宏定义、类型定义、函数、变量申明
  • 编译：编译器对每个源文件进行编译生成机器指令（编译包含头文件的源文件时，会自动编译头文件）——》目标文件：.o或.obj。每个源文件对应一个目标文件（很多个目标模块）
  • 链接：链接器把编译器生成的目标代码和依赖的任何其他代码（函数调用中调用的库函数）整合起来（连接各个模块），最终生成可执行文件

GCC编译器一步构建：

  • 为了易于构建大型程序，产生了makefile的概念

    • makefile不但列出属于程序的文件，而且描述文件间的依赖关系

    • ```
      justify： justify.o word.o line.o

      gcc -o justify justify.o word.o line.o

      justify.o： justify.c word.h line.h

      gcc -c justify.c

      word.o： word.c word.h

      gcc -c word.c

      line.o： line.c line.h

      gcc -c line.c

      make 目标

  • 结构的每一个成员单独占用空间，且采取内存对齐

  • 联合：编译器只为其中最大的成员分配内存空间，其他成员共享。能极大的节省内存空间。

十一、sizeof易错点

  • sizeof(数组变量)=数组大小
  • sizeof(指针变量)=指针大小（依操作系统而定）

十二、常用！！memset和memcpy

1. memcpy

  • 以source指向的地址为起点，将连续的n个字节数据，复制到以destin指向的地址为起点的内存中。
  • 常用于字符串数组中字符的修改
  • eg：memcpy(buf+4, &len_n, 4);

2. memset

  • 初始化函数，作用是将某一块内存中的全部设置为指定的值
  • eg：memset(buf,0,sizeof(buf));
