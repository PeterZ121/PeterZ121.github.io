分治算法

基本思想

​ “分而治之”,把一个复杂的问题分成两个或更多的相同或相似的子问题，再把子问题分成更小的子问题……直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并

经典问题

基本步骤

  1. 分解：将原问题分解为若干个规模较小，相互独立，与原问题形式相同的子问题
  2. 解决：若子问题规模较小而容易被解决则直接解，否则递归地解各个子问题
  3. 合并：将各个子问题的解合并为原问题的解。

汉诺塔

汉诺塔问题源自印度一个古老的传说，印度教的“创造之神”梵天创造世界时做了 3 根金刚石柱，其中的一根柱子上按照从小到大的顺序摞着 64 个黄金圆盘。梵天命令一个叫婆罗门的门徒将所有的圆盘移动到另一个柱子上，移动过程中必须遵守以下规则：

  • 每次只能移动柱子最顶端的一个圆盘；
  • 每个柱子上，小圆盘永远要位于大圆盘之上；

汉诺塔游戏的演示和思路分析:

  1. 如果是有一个盘， 起始柱->最终柱

  2. 如果我们有 n >= 2 情况，我们总是可以看做是两个盘 1.最下边的盘 2. 上面的盘

    • 先把上面的所有盘 起始柱->过渡柱
    • 把最下边的盘 起始柱->最终柱
    • 把B塔的所有盘 过渡柱->最终柱
  3. 分解的原子问题：两个盘
    • 小盘：起始柱->过渡柱
    • 大盘：起始柱->最终柱
    • 小盘：过渡柱->最终柱

由此可知：

​ 设盘数为n，则汉诺塔执行次数为2^n^-1.

动态规划算法

基本思想

​ 与分治算法类似：将求解问题分解为若干子问题。但不同的是：动态规划划分的子问题不是互相独立的->下一个子阶段的求解建立在上一个子阶段的解的基础上，进一步求解

​ 通过填表的方式逐步推进，得到最优解

背包问题

一、01背包问题：装入物品不能重复

思路分析和图解

​ 每次遍历到的第i个物品，根据w[ i ]和v[ i ]来确定是否需要将该物品放入背包中。即对于给定的n个物品，设v[ i ]、w[ i ]分别为第i个物品的价值和重量，C为背包的容量。再令value[ i ] [ j ]表示在前i个物品中能够装入容量为j的背包中的最大价值。则我们有下面的结果：

  1. value[ i ] [ 0 ]=0（背包容量为0时，装入价值为0）；value[ 0 ] [ j ]=0（0个物品装入背包价值为0）;

  2. 当w[i]> j 时 不能装入第i个物品：value[ i ] [ j ]=value[ i-1 ] [ j ] ； 当准备加入新增的商品的容量大于当前背包的容量时，说明无法装入该背包，就直接使用上一个单元格（ i - 1 ）的装入策略

  3. 当w[i]<=j时 能装入第i个物品 考虑应不应该放该物品 ：value[ i ] [ j ]=max{value[ i - 1 ] [ j ], v[ i ] + value[ i - 1 ] [ j - w [ i ] ]} ；当准备加入新增的商品的容量小于等于当前背包容量时（说明可以尝试装入背包），比较两个装入模式：

    • 上一个单元格的装入策略的价值 value[ i - 1 ] [ j ]
    • 当前商品价值+装入当前商品后剩余空间的装入策略的价值（之前的最优解） v[ i ] + value[ i - 1 ] [ j - w [ i ] ]

  • 即是：某容量背包==不装入该物品==的价值和==在装入该物品且剩余空间实行之前最优解==的价值的比较

    最后，二维数组的最后一个元素即为最优解

二、非01背包问题：装入物品可重复

物品 重量 价值
1 2 1
2 3 3
3 4 5
4 5 6
算法思路

  • 与01背包问题总体思路相同，区别点在：

    • value[ i ] [ j ]表示在前i==种==物品中能够装入容量为j的背包中的最大价值
    • 若能放入某种物品，还要多考虑：==放多少个才能实现装入的最大价值==

  • 有以下结果：

    • value[ i ] [ 0 ]=0（背包容量为0时，装入价值为0）；value[ 0 ] [ j ]=0（0个物品装入背包价值为0）;

    • 当w[i]> j 时 不能装入第i种物品：value[ i ] [ j ]=value[ i-1 ] [ j ] ； 当准备加入新增的商品的容量大于当前背包的容量时，说明无法装入该背包，就直接使用上一个单元格（ i - 1 ）的装入策略

    • 当w[i]<=j时 可装入第i种物品，并且最多可以放j/w[i]个 考虑应该放多少个以实现最大价值：

      • 放0个：x0 = value[ i - 1 ] [ j ]

      • 放1个：x1 = v[ i ] + value[ i - 1 ] [ j - w[ i ] ]

      • 放2个：x2 = 2v[ i ] + value[ i - 1 ] [ j - 2w[ i ] ]

      • …………

      • 放n个（n <= j / w[ i ]）：xn = n v[ i ] + value[ i - 1 ] [ j - n w[ i ] ]

      • 最后value[ i ] [ j ] = max(x0, x1, …………, xn)

最后，二维数组的最后一个元素即为最优解

贪心算法

贪婪算法(贪心算法)是指在对问题进行求解时，在每一步选择中都采取最好或者最优(即最有利)的选择，从而希望能够导致结果是最好或者最优的算法

贪婪算法所得到的结果不一定是最优的结果(有时候会是最优解)，但是都是相对近似(接近)最优解的结果

与动态规划不同点在于：动态规划每一次最优解都基于上一步的最优解，环环相扣；而贪婪算法每一步，则是在该步骤中的所有选择中找到最优解并执行，是独立的

经典题目——集合覆盖

假设存在如下表的需要付费的广播台，以及广播台信号可以覆盖的地区。 如何选择最少的广播台，让所有的地区都可以接收到信号

广播台 覆盖地区
K1 “北京”, “上海”, “天津”
K2 “广州”, “北京”, “深圳”
K3 “成都”, “上海”, “杭州”
K4 “上海”, “天津”
K5 “杭州”, “大连”

思路分析

​ 目前并没有算法可以快速计算得到准备的值， 使用贪婪算法，则可以得到非常接近的解，并且效率高。选择策略上，因为需要覆盖全部地区的最小集合:

  • 遍历所有的广播电台, 找到一个覆盖了最多未覆盖的地区的电台(此电台可能包含一些已覆盖的地区，但没有关系）
  • 将这个电台加入到一个集合中(比如ArrayList)(select), 想办法把该电台覆盖的地区在下次比较时去掉。
  • 重复第1步直到覆盖了全部的地区
